# Фильтр желтушных новостей

Сервис для автоматической оценки «желтушности» новостных статей. Анализирует текст на наличие эмоционально окрашенной лексики (позитивной или негативной) и вычисляет рейтинг — чем выше процент заряженных слов, тем более «жёлтой» считается статья.

## Как это работает

1. Пользователь (или браузерный плагин) отправляет на сервер список URL‑адресов статей.
2. Сервер скачивает HTML каждой страницы, извлекает текст статьи с помощью адаптера под конкретный новостной сайт.
3. Текст разбивается на слова с учётом морфологии (библиотека `pymorphy3`), приводятся к нормальной форме.
4. Подсчитывается доля слов, входящих в словарь «заряженных» слов (негативных или позитивных). Результат возвращается в виде JSON.

### Поддерживаемые сайты
Пока реализован адаптер только для [ИНОСМИ.РУ](https://inosmi.ru/). Для добавления нового сайта нужно создать собственный адаптер в папке `adapters/`. Адаптер должен содержать функцию `sanitize(html)`, которая извлекает текст статьи из HTML и, при невозможности, выбрасывает исключение `ArticleNotFound`.

# Как установить

Вам понадобится Python версии 3.7 или старше. Для установки пакетов рекомендуется создать виртуальное окружение.

Первым шагом установите пакеты:

```python3
pip install -r requirements.txt
```

# Как запустить
## Консольная версия (main.py)
Используется для отладки и тестирования алгоритма на фиксированном наборе статей. Запускается командой:
```python3
python main.py
```
В терминал выводится рейтинг каждой статьи из списка TEST_ARTICLES (задан внутри файла).
## Веб‑сервер (server.py)
Запускает API, доступное по адресу http://127.0.0.1:8080. Сервер обрабатывает GET‑запросы с параметром urls (список URL, разделённых запятыми).

```
python server.py
```
Пример запроса:
```
http://127.0.0.1:8080/?urls=https://inosmi.ru/20260629/ormuz-279091044.html,https://lenta.ru/brief/2021/08/26/afg_terror/
```
Пример ответа (JSON):
```

```
Возможные статусы:

OK – статья успешно проанализирована.

FETCH_ERROR – ошибка при скачивании (неверный URL, недоступный сайт).

PARSING_ERROR – адаптер не смог извлечь текст (сайт не поддерживается или изменилась разметка).

TIMEOUT – превышено время ожидания ответа от сайта (5 секунд).

ERROR – прочие ошибки (например, сбой морфологического анализатора).

Ограничения:

Максимальное количество URL в одном запросе – 10 (защита от DDoS).

Таймаут на скачивание каждой страницы – 5 секунд.


# Как запустить тесты

Для тестирования используется [pytest](https://docs.pytest.org/en/latest/), тестами покрыты фрагменты кода сложные в отладке: text_tools.py и адаптеры. Команды для запуска тестов:

```
python -m pytest adapters/inosmi_ru.py
```

```
python -m pytest text_tools.py
```
